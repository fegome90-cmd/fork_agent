## Clasificación de documentos en docs/investigacion/

Este README indica la categorización propuesta para los archivos existentes en docs/investigacion/, con el objetivo de enfocar la revisión y la preservación de información relevante.

| Archivo | Categoría | Razón de la recomendación |
|---------|-----------|-----------------------------|
| ECC.md | ARCHIVE | Documento conceptual sobre bloques de Claude Code y habilidades; útil como referencia histórica, pero no esencial para la arquitectura actual ni para el desarrollo inmediato. |
| brain_dope.md | ARCHIVE | Notas de investigación/Brain Dope; contexto útil, pero no crítico para los flujos de trabajo actuales. |
| brain_dope_work_O.md | ARCHIVE | Detalle de planificación de work_O; relevante solo para auditoría histórica, no para la implementación actual. |
| kernel.md | KEEP | Descripción de la arquitectura de orquestación (V4) y plan de integración; núcleo para entender cómo se conectan los componentes. |
| oficial_doc.md | KEEP | Índice/documentación oficial de Claude Code; referencia clave para prácticas recomendadas y conceptos operativos. |
| diff.md | KEEP | Comparación entre MCP de referencia y fork_agent; guía para posibles migraciones/integraciones y evaluación de alcance. |

## Priorización de KEEP (si hay más de 3 archivos marcados como KEEP)
- Los 3 archivos más importantes para conservar como KEEP son: kernel.md, oficial_doc.md y diff.md.
- Razón: proporcionan la visión arquitectónica, la referencia de prácticas oficiales y el marco de evaluación/integra­ción entre MCP y fork_agent, que son críticos para comprender y avanzar en el proyecto.

## Notas
- Si se requieren cambios en el orden de preservación o en las categorías, indícalo y ajusto la clasificación.
