## Issues Críticos (DEBEN ARREGLARSE)

### 1. Tests de Error Handling - `tests/unit/infrastructure/test_observation_repository.py`

**Problema**: Los tests intentan mockear `__enter__` de `DatabaseConnection`, pero este atributo es read-only porque la clase usa `__slots__`.

```python
# Lines 652-693 - Error handling tests
with patch.object(db_connection, "__enter__") as mock_enter:
    # AttributeError: 'DatabaseConnection' object attribute '__enter__' is read-only
```

**Solución**: Usar un enfoque alternativo para testear error handling:
1. Crear un mock de `DatabaseConnection` completo
2. Usar `side_effect` en `execute` para simular errores
3. O usar un fixture que inyecte una conexión que falle

**Código sugerido**:
```python
def test_create_handles_database_error(self, tmp_path: Path) -> None:
    db_path = tmp_path / "test.db"
    config = DatabaseConfig(db_path=db_path)
    
    # Create a connection that will fail
    class FailingConnection(DatabaseConnection):
        def __enter__(self):
            conn = super().__enter__()
            # Override execute to raise error
            original_execute = conn.execute
            def failing_execute(*args, **kwargs):
                if "INSERT" in str(args[0] if args else ""):
                    raise sqlite3.Error("Database error")
                return original_execute(*args, **kwargs)
            conn.execute = failing_execute
            return conn
    
    repo = ObservationRepository(FailingConnection(config))
    observation = Observation(id="test", timestamp=1, content="test")
    
    with pytest.raises(RepositoryError):
        repo.create(observation)
```
